## 💡 重要提示

### 1. 不需要对接的接口
以下接口属于**管理后台功能**，小程序端**不需要对接**：
- 管理员认证相关（8个）
- 设备管理相关（5个）
- 客户管理相关（5个）
- 订单管理相关（3个）
- 绑定管理相关（7个）
- 文件管理相关（3个）
- 邮件服务相关（2个）
- 套餐管理相关（3个）
- 账单管理相关（4个）
- 发票管理相关（3个）
- Banner和推文管理（7个）
- 统计和日志相关（2个）
- 管理员支付相关（4个）
- 支付回调接口（3个，由支付平台调用）

### 2. 接口封装建议
- 创建统一的请求封装层 `utils/request.js`
- 创建API接口定义文件 `utils/api.js`
- 统一处理token认证、错误处理、loading状态

### 3. 数据缓存策略
建议缓存的数据：
- 套餐列表（1小时）
- 省市数据（永久）
- Banner列表（30分钟）
- 用户信息（登录期间）

### 4. 错误处理
- 统一处理网络错误
- Token过期自动跳转登录页
- 提供友好的错误提示
